Unit-1: Database Management System

Data

According to the Oxford Dictionary "Data is distinct pieces of information, usually formatted in a special way".
Data can be measured, collected, reported, and analyzed, whereupon it is often visualized using graphs, images, or other analysis tools. Raw data ("unprocessed data") may be a collection of numbers or characters before it's been "cleaned" and corrected by researchers.

Information

Information is data that has been processed, organized, or structured in a way that makes it meaningful, valuable, and useful. It is data that has been given context, relevance, and purpose. It gives knowledge, understanding, and insights that can be used for decision-making, problem-solving, communication, and various other purposes.

Features of Information

  1. Subjectivity
    2. Characteristics of information refer to the qualities that define its usefulness. Subjectivity is one such trait, which denotes the extent to which the information has been influenced by personal opinion or bias.
  2. Relevance
    3. The relevance of information is another important characteristic. It implies that the collected data should be useful and applicable to the purpose for which it is being used. For instance, a student searching for information about a particular topic should have access to data that is related and helpful in furthering their understanding and research.
  3. Timeliness
    4. Timeliness is an important characteristic of information because it provides evidence about when the data was collected or updated last.
  4. Accuracy
    5. The accuracy of information is one of its most important characteristics. This implies that the content should provide correct and reliable data which has been verified by reliable sources.
  5. Correct Information Format
    6. Correct information format means ensuring that the data provided is structured in a way that makes it easy for users to access, understand, and utilize it. This involves presenting the data in an organized manner so that it can be read and interpreted quickly without any confusion or ambiguity.

Terminologies in Database

Table, Field, Record, Tuple, Object, Keys

Table: A collection of related data organized in rows and columns in a database.
Field: A single piece of data or attribute in a table column (e.g., Name, Age).
Record: A complete set of fields representing one item or entity (a row in a table).
Tuple: Another term for a record; a row in relational databases.
Object: In object-oriented databases, an instance containing both data (fields) and methods (functions).
Keys: Special fields used to uniquely identify records (e.g., Primary Key) or link tables (Foreign Key).

Data Dictionary

A centralized repository that stores metadata about the database structure, such as definitions of tables, fields, data types, and relationships.
Helps users and administrators understand and manage the database schema.

Database Management System (DBMS)

Introduction: Software that allows users to define, create, maintain, and control access to databases.
Manages data efficiently, supports data manipulation, and ensures data integrity and security.
Objectives
  • Provide data abstraction and independence.
  • Facilitate efficient data storage, retrieval, and update.
  • Enforce data integrity and security.
  • Support concurrent access and recovery from failures.
Advantages
  1. Reduced data redundancy and inconsistency.
  2. Improved data sharing and security.
  3. Centralized data management.
  4. Better data integrity and backup/recovery features.
Disadvantages
  1. High initial cost for setup and training.
  2. Complexity in design and management.
  3. Performance overhead due to abstraction layers.
  4. Risk of data loss or corruption if not properly managed.

Types of Database Model

  • Hierarchical: Data organized in a tree-like structure with parent-child relationships. Fast for simple, one-to-many data.
  • Network: Complex graph-like structure allowing many-to-many relationships via sets.
  • Relational: Data stored in tables (relations) with rows (tuples) and columns (fields). Most widely used.
  • Entity-Relationship (ER): Conceptual model representing data as entities, attributes, and relationships, often used for database design.

Integrity Constraints and Types:

  • Domain: Ensures data in a field conforms to a specific data type or range.
  • Entity Ensures each entity (record) is uniquely identifiable (usually via primary key).
  • Referential: Maintains consistency between related tables through foreign keys.
  • Keys: Enforces uniqueness and relationships; includes primary keys, foreign keys, and candidate keys.

Normalization

Introduction:

Process of organizing data to reduce redundancy and improve data integrity.

Normal Forms

1NF (First Normal Form): Eliminate repeating groups; atomic values only.
2NF (Second Normal Form): Remove partial dependency on a composite primary key.
3NF (Third Normal Form): Remove transitive dependency; non-key fields depend only on the primary key.

Advantages:

  • Eliminates data redundancy
  • Improves data integrity
  • Simplifies data maintenance.

Disadvantages:

  • Can lead to complex queries and joins.
  • Might affect performance due to multiple tables.

Centralized and Distributed Database

Introduction:

Centralized DB: Data stored and managed at a single location.

Distributed DB: Data spread across multiple physical locations interconnected via a network.

Advantages:

  • Centralized: Easier management and control.
  • Distributed: Improved reliability, scalability, and faster local access.

Disadvantages:

  • Centralized: Single point of failure, limited scalability.
  • Distributed: Complex design, synchronization issues, higher cost.

Comparison:

Aspect Centralized Distributed
Location Single site Multiple sites
Reliability Lower (single failure) Higher (redundancy)
Scalability Limited High
Complexity Simpler Complex

Database Security

Introduction:

Protecting database from unauthorized access, misuse, or damage.

Challenges:

  • Unauthorized access.
  • Data breaches.
  • Maintaining data integrity.
  • Managing user privileges.

Security Measures:

  • Authentication and authorization.
  • Encryption.
  • Backup and recovery plans.
  • Auditing and monitoring access.

Roles of DBA (Database Administrator):

  • Manage and maintain database systems.
  • Implement security policies.
  • Ensure data integrity and availability.
  • Backup and recovery management.
  • Performance tuning and user support.
