Unit-1: Database Management System

Data

According to the Oxford Dictionary "Data is distinct pieces of information, usually formatted in a special way."
Data can be measured, collected, reported, and analyzed, and often visualized using graphs, images, or tools. Raw data may be a collection of numbers or characters before it’s been cleaned and corrected by researchers.

Information

Information is data that has been processed, structured, or organized to be meaningful and useful. It provides insights, supports decision-making, and enables problem-solving.

Features of Information

  1. Subjectivity: Influence from personal opinion or bias.
  2. Relevance: Data should be applicable and purposeful.
  3. Timeliness: Indicates when data was last updated.
  4. Accuracy: Data should be correct and verified.
  5. Correct Format: Information must be structured clearly for interpretation.

Terminologies in Database

Table, Field, Record, Tuple, Object, Keys

Table: A collection of related data in rows and columns.
Field: A single data item (column).
Record: A complete row representing one entity.
Tuple: Another term for a record.
Object: In object-oriented DBs, includes both fields and functions.
Keys: Unique identifiers like primary or foreign keys.

Data Dictionary

A centralized repository storing metadata about database structure—definitions of tables, fields, data types, and relationships.

Database Management System (DBMS)

Introduction: Software used to define, create, manage, and control access to databases.

Objectives

  • Provide data abstraction and independence.
  • Facilitate efficient data operations.
  • Ensure integrity and security.
  • Support concurrent access and recovery.

Advantages

  1. Reduced redundancy and inconsistency.
  2. Improved sharing and security.
  3. Centralized control.
  4. Enhanced integrity and recovery.

Disadvantages

  1. High setup cost.
  2. Design complexity.
  3. Performance overhead.
  4. Risk of data loss.

Types of Database Model

  • Hierarchical: Tree-like structure, parent-child relationships.
  • Network: Graph structure allowing many-to-many links.
  • Relational: Table-based format using tuples and fields.
  • Entity-Relationship (ER): Conceptual model using entities and relationships.

Integrity Constraints and Types

  • Domain: Validates data type and range.
  • Entity: Ensures unique identity via keys.
  • Referential: Maintains table relations with foreign keys.
  • Keys: Enforce uniqueness and relational links.

Normalization

Introduction

Organizing data to minimize redundancy and maintain integrity.

Normal Forms

1NF: Atomic values only; no repeating groups.
2NF: Removes partial dependency on composite keys.
3NF: Eliminates transitive dependency; non-key fields depend solely on the primary key.

Advantages

  • Reduces redundancy
  • Enhances integrity
  • Simplifies maintenance

Disadvantages

  • Complex queries
  • Performance impact due to multiple joins

Centralized and Distributed Database

Introduction

Centralized DB: Stored at one location.
Distributed DB: Spread across interconnected sites.

Advantages

  • Centralized: Easier control and management.
  • Distributed: Better reliability and access speed.

Disadvantages

  • Centralized: Single point of failure.
  • Distributed: Design complexity, synchronization challenges.

Comparison

Aspect Centralized Distributed

Database Security

Introduction

Database security focuses on safeguarding data from unauthorized access, manipulation, or destruction while preserving confidentiality, integrity, and availability.

Challenges

  • Preventing unauthorized access and breaches
  • Maintaining consistency and accuracy of data
  • Managing diverse user roles and privileges
  • Handling internal threats and accidental misuse

Security Measures

  • Strong authentication and role-based authorization
  • Data encryption during storage and transmission
  • Routine backups and disaster recovery planning
  • Audit trails and real-time activity monitoring

Roles of DBA (Database Administrator)

  • Installing and configuring DBMS software
  • Defining and enforcing access controls
  • Maintaining system performance and health
  • Handling data recovery and backup schedules
  • Providing support to users and developers

3. Differentiate Various DBMS Models

Hierarchical Model

Uses parent-child relationships to structure data like a tree. It's fast for retrieving structured data but rigid and hard to modify.

Network Model

Represents data using a graph with multiple parent-child links. It’s flexible for complex relationships but challenging to manage and design.

Relational Model

Stores data in tables with rows and columns. It supports SQL, is scalable, and user-friendly—but may be slower for highly complex relationships.

Object-Oriented Model

Combines database features with object-oriented programming principles. Ideal for multimedia and complex data handling but less widely adopted and harder to implement.

4. What is a Relational Database? How is it Different from Other Models?

Answer:

A relational database stores data in structured tables (relations) made up of rows and columns. Each table represents a distinct entity, and relationships are established through primary and foreign keys.

Key Differences from Other Models:

  • Hierarchical Model: Organizes data in a strict tree-like structure; relational model offers flexible table relationships.
  • Network Model: Supports many-to-many parent-child links but is more complex than relational tables.
  • Object-Oriented Model: Stores data as objects, blending code and data; relational model uses clearly defined schemas and tables.

Advantages of Relational Databases:

  • Uses SQL for powerful and standardized querying
  • Scalable and easier to maintain
  • Supports normalization and ensures data integrity

The relational model's blend of simplicity, flexibility, and strong theoretical foundation makes it the most widely adopted approach in modern data systems.

5. What is Data Redundancy? How Does DBMS Help Reduce It?

Answer:

Data redundancy refers to the unnecessary repetition or duplication of data within a system, which can lead to inconsistencies, higher storage requirements, and difficulties in maintaining accuracy.

A DBMS reduces redundancy by organizing related data into separate, well-structured tables and linking them using keys (like primary and foreign keys). This centralized management allows updates to be made in one place without duplicating the same information elsewhere—ensuring consistency and efficient use of resources.

8. Define Normalization. Explain 1NF, 2NF and 3NF with Examples

Answer:

Normalization is the process of structuring a database to reduce redundancy and enhance data integrity. It involves dividing large tables into smaller, more manageable ones and defining relationships between them.

First Normal Form (1NF)

Ensures that each column contains atomic values and each record is unique.
Example: A field containing multiple phone numbers violates 1NF; it's corrected by storing each phone number in a separate row.

Second Normal Form (2NF)

Achieved when the table is in 1NF and all non-key attributes depend fully on the entire primary key.
Example: If a table with a composite key has fields that depend on only part of the key, those dependencies are moved to a new table.

Third Normal Form (3NF)

Achieved when the table is in 2NF and all attributes depend only on the primary key, not on other non-key attributes.
Example: Removing a column like "Department Location" that depends on "Department Name" rather than directly on the primary key.

Through these steps, normalization ensures that data is logically stored, reduces duplication, and simplifies future updates and queries.

9. Explain Primary Key, Foreign Key, and Candidate Key with Examples

Answer:

A primary key is a field or combination of fields in a table that uniquely identifies each record. It must contain unique values and cannot be null.
Example: In the Students table, student_id serves as the primary key since it uniquely identifies each student.

A foreign key is a field in one table that refers to the primary key in another, establishing a link between the two tables.
Example: In the Enrollments table, student_id acts as a foreign key referencing Students.student_id.

A candidate key is any field or set of fields that can uniquely identify records in a table. A table may have multiple candidate keys, but only one is chosen as the primary key.
Example: In the Students table, both student_id and email could be candidate keys, but student_id is selected as the primary key.

10. What is SQL? Explain Its Components and Common Functions

Answer:

SQL (Structured Query Language) is the standard language used to manage and manipulate data in relational databases. It enables users to define database structures, perform data operations, and control access.

SQL Components:

  • DDL (Data Definition Language): Includes commands like CREATE, ALTER, and DROP to define and modify database structures.
  • DML (Data Manipulation Language): Enables data changes through commands like INSERT, UPDATE, and DELETE.
  • DQL (Data Query Language): Mainly uses SELECT to retrieve data from tables.
  • DCL (Data Control Language): Manages access permissions using GRANT and REVOKE.

Common SQL Functions:

  • Aggregate Functions: AVG(), SUM(), COUNT(), MIN(), MAX() — used for mathematical computations.
  • String Functions: CONCAT(), UPPER(), LOWER(), SUBSTRING() — used for text manipulation.
  • Date Functions: NOW(), DATEADD(), DATEDIFF() — used for handling date and time values.

SQL is essential for effective communication with relational databases, providing both power and flexibility for data management.

12. Define the Following Terms

  1. Data Dictionary
  2. Primary Key
  3. Relationship
  4. Data Manipulation Language (DML)
  5. Structured Query Language (SQL)
  6. Data Integrity
  7. Data Definition Language (DDL)
  8. Data Security
  9. Database System

Answer:

  • Data Dictionary: A centralized repository containing metadata about the database, including definitions of tables, columns, data types, and relationships.
  • Primary Key: A unique identifier assigned to each record in a table, such as student_id in a Students table.
  • Relationship: A logical connection between tables based on shared fields, typically established using primary and foreign keys.
  • Data Manipulation Language (DML): A subset of SQL used for managing data in tables using commands like INSERT, UPDATE, and DELETE.
  • Structured Query Language (SQL): The standard language used to define, manipulate, query, and control access in relational databases.
  • Data Integrity: The accuracy, reliability, and consistency of data over its entire lifecycle within the database.
  • Data Definition Language (DDL): SQL commands used to define or modify database structures, such as CREATE, ALTER, and DROP.
  • Data Security: Strategies and controls put in place to protect data from unauthorized access, theft, or corruption.
  • Database System: A coordinated combination of hardware, software, data, and procedures that stores and manages digital information efficiently.
